Wholesale Packaging Preferences — Build Notes
==============================================
Build date: 2025

## What was built

### Backend (server/routes.ts)
Three new routes added near the end of registerRoutes (before the closing brace),
after the existing flex/auth routes, reusing the existing FLEX_BASE and FLEX_HEADERS
constants already defined in scope:

1. GET /api/wholesale/customers
   - Fetches wholesale customers from Flex API (customer_group_id=13f392c3-fa06-417e-870a-47912a3afc78)
   - Merges with existing records from Supabase wholesale_packaging_prefs table
   - Returns alphabetically sorted array with all pref fields as camelCase
   - Falls back to 500 if Flex API fails

2. GET /api/wholesale/active-customers
   - Fetches orders from last 90 days for the same customer group
   - Returns array of unique customer UUIDs
   - Returns empty array [] on any failure (non-breaking)

3. POST /api/wholesale/prefs
   - Accepts: flexCustomerId, flexCustomerNumber, companyName, paper, wrapStyle,
     allItemsGreaseproof, barcodeLabels, specialNotes
   - Resolves staff name from req.session.staffId → Supabase staff table
   - Upserts into wholesale_packaging_prefs on flex_customer_id conflict
   - Returns saved record as camelCase

## Supabase table required
The wholesale_packaging_prefs table must exist in Supabase with this schema:
  CREATE TABLE wholesale_packaging_prefs (
    id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    flex_customer_id text UNIQUE NOT NULL,
    flex_customer_number integer,
    company_name text,
    paper text,           -- 'branded' | 'plain_white'
    wrap_style text,      -- 'open' | 'burrito'
    all_items_greaseproof boolean,
    barcode_labels boolean,
    special_notes text,
    updated_at timestamptz,
    updated_by text
  );
If this table doesn't exist yet, it needs to be created before the routes will work.
